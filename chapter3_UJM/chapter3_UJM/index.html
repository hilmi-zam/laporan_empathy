Konsep Inti

Apa itu User Persona?

Representasi fiktif, namun berbasis data nyata, dari segmen pengguna target (target user) Anda.

Mitos Persona
  • Dibuat berdasarkan tebakan atau asumsi tim developer.
  • Hanya representasi 1 orang di dunia nyata.
  • Fokus hanya pada umur dan pekerjaan (demografi tok).
Fakta Persona
  • Dibuat dari pola perilaku sekumpulan orang (hasil wawancara).
  • Sintesis dari banyak user nyata.
  • Fokus pada Tujuan (Goals) dan Rasa Sakit (Pain Points).
CDIO Framework
Conceive:
Definisikan Masalah.

Dalam tahapan Conceive (Merumuskan), Persona bertindak sebagai jangkar. Jika tim berdebat soal fitur, tanyakan: "Apakah 'Budi' (Persona kita) akan menggunakan fitur ini?"

Anatomi User Persona yang Solid

  • Name & Bio: Nama fiktif, kutipan (quote) khas, profesi, umur.
  • Goals / Needs: Apa tujuan akhir yang ingin mereka capai? (Misal: "Lulus KRS tanpa error").
  • Frustrations / Pain Points: Apa yang membuat mereka kesal saat ini?
  • Behaviors: Kebiasaan sehari-hari menggunakan teknologi (Mobile/Desktop? Sosmed?).
Persona tanpa "Pain Point" tidak bisa memberikan solusi desain!

User Journey Mapping (UJM)

Persona memberi tahu kita SIAPA pengguna kita. Journey Mapping memberi tahu kita BAGAIMANA pengalaman mereka langkah demi langkah.

Definisi

Visualisasi dari proses yang dilalui pengguna untuk mencapai suatu tujuan khusus pada produk/layanan Anda.

  • Menemukan "Gaps" (Celah) di mana user kebingungan.
  • Memecah proses kompleks menjadi fase-fase kecil.
  • Memberi fokus pada emosi pengguna di setiap titik interaksi (Touchpoint).
Visualizing the Journey across touchpoints
CDIO Framework
Design:
Peta Pengalaman.

Dalam tahapan Design, UJM menjadi cetak biru (blueprint). Anda tidak bisa merancang layar aplikasi (UI) yang baik jika Anda tidak tahu urutan kronologis kejadian (Journey).

Anatomi Journey Map

Elemen wajib dalam pemetaan perjalanan pengguna:

1. Actor & Scenario

Siapa? (Persona) dan Apa tujuannya? (Misal: Tegar ingin KRS-an).

2. Phases (Fase Berjalan)

Contoh: Persiapan -> Eksekusi KRS -> Konfirmasi Dosen.

3. Actions & Touchpoints

Apa yang user KLIK? Gadget apa yang dipakai? Interaksi sistem vs manusia.

4. Emotion & Thoughts

Kurva emosi (Senang, Bingung, Frustrasi) + Apa yang dipikirkan.

Action: Login siakad, cek jadwal matkul.

Thought: "Semoga kelas UX pagi belum penuh."

Phase 2: Eksekusi (The Pain)

Thought: "Waduh, koneksi atau servernya nih?!"

🤬
Phase 3: Konfirmasi

Action: Refresh 5x, akhirnya kelas masuk, chat Dosen PA via WA.

Thought: "Huft, akhirnya selamat meskipun deg-degan."
